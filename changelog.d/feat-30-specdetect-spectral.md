### Added

**`specdetect_audit` — SpecDetect spectral descriptors of the surprisal vector (spec 30, M1; new `discrimination_spectral` surface).** A new Surface-5 discrimination signal that reads the *sequential/spectral structure* of the per-token surprisal vector SETEC already pays a forward pass for — the shape over time of how predictability rises and falls, beyond the single lag-k autocorrelation the DivEye point moments (`surprisal_audit`) capture. Takes the log-probability sequence (`-bits` of what `score_text` already returns — no new backend method, no second forward pass), removes its mean, and computes a real-input DFT magnitude spectrum, reading off `spectral_centroid`, `low_freq_energy_frac`, `spectral_flatness`, `peak_frequency_bin` (+ normalized magnitude), and a human-readable `dominant_period_tokens`. Clean-room reimplementation of **SpecDetect** from Yang et al. 2025, *SpecDetect: Spectral Analysis for Robust and Efficient Zero-Shot Detection of LLM-Generated Text* ([arXiv:2508.11343](https://arxiv.org/abs/2508.11343)). The DFT is a stdlib `cmath` real-DFT (the path CI exercises); `numpy.rfft` is an optional fast path with a parity fixture, so `import`ing the module pulls no torch and no numpy and the whole M1 core runs model-free over an injected sequence.

Posture (mirrors `fast_detect_curvature`'s "uncalibrated, no verdict"): the surface emits the descriptor values + a PROVISIONAL band, with **no** `is_ai` / `ai_probability` / `verdict` / composite `score` field. The band *values* name the measured spectrum property — `flat-spectrum` / `concentrated-spectrum` / `indeterminate` (the `surprisal_audit` `smoothed`/`typical` discipline) — never a machine/AI/human class; a band that named the inference target would be one rename from a verdict (review P1, folded). A `≥2-of-N` descriptor vote so no single descriptor decides; the band always carries `provisional: True` + `calibration_anchor: user-baseline-required` + `thresholds_used`. Formulaic, translated, and ESL prose produce regular spectra too — a concentrated spectrum is **not** an AI finding and a flat one is **not** a human finding (absence is not evidence; the surface emits no "human" label any more than an "AI" one). The spectrum is tokenizer-/model-bound and ships **no** calibrated threshold; the operator adjudicates via the existing calibration pipeline. Degenerate inputs (constant / empty / sub-`MIN_SERIES_FOR_SPECTRUM`) return descriptors `None` (or flagged) with a matching caveat, never a spurious number.

Orthogonality is asserted structurally (`test_orthogonal_statistic`, the curvature shape): the spectral code references no `mean_surprisal`, `sd_surprisal`, `autocorrelation`/`lag_` field, `curvature`, or Binoculars ratio — it reads only the raw sequence (a comment- and string-literal-stripped source scan, so the docstring may name the forbidden symbols as posture documentation). By construction a single lag-k ACF is one projection of the second-order structure while the magnitude spectrum is that structure whole (Wiener–Khinchin).

**Lastde / Lastde++ ships as a tested library function only, NOT wired into the surface at M1.** `diversity_entropy` (ordinal-pattern / permutation-entropy primitive) and `lastde_multiscale` (the Lastde++ multi-scale aggregate) are implemented and unit-tested — a strictly monotone ramp → diversity entropy ≈ 0, an i.i.d. shuffle → ≈ 1 — but whether Lastde earns a place beside SpecDetect is decided by the M2 in-tree **orthogonality gate** against the DivEye moments + curvature (the spec's load-bearing question: `specs/00` pre-flagged the DivEye-ACF overlap as "marginal"). The `audit()` surface output emits SpecDetect only and surfaces `lastde_block_gated_to_m2_orthogonality_check`. Rooted in Xu et al. 2024, *Training-free LLM-generated Text Detection by Mining Token Probability Sequences* ([arXiv:2410.06072](https://arxiv.org/abs/2410.06072)).

A `feat` (MINOR): one new capability (`specdetect_audit`) + one new task-surface label (`discrimination_spectral`), via the drop-in `capabilities.d/` fragment + its per-id `_golden_capabilities/specdetect_audit.json` + the `claim_license_surfaces/discrimination_spectral.txt` fragment (the review P1 build fix — without it `build_output` hard-raises on the unregistered surface). Both arXiv roots are cited in the claim license, this fragment, and the PR body (the fleet cite-arXiv rule).
