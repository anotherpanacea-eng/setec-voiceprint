### Added

**`function_word_adjacency_audit` — the function-word adjacency NETWORK (spec 32; a new capability on the existing `voice_coherence` surface, no new surface).** Models the target's function-word transitions as a directed graph (nodes = the 135 canonical `FUNCTION_WORDS`, edges = adjacent-in-text transition frequencies) and emits graph-structural descriptors the flat-bigram view cannot express: node centralities (in/out-degree + a **PageRank** stationary centrality via numpy power iteration), per-node and global transition entropy, directionality (reciprocity / weight asymmetry), network density, and small directed motifs (2-cycles, length-3 directed walks, self-loops). This is the **structure** read of the SAME function-word transitions `function_word_grammar_audit` reads as flat top-20 bigrams — the grammar audit owns the bigram COUNTS, FWAN owns the GRAPH READ. A grep-proven mechanical disjointness (centrality / per-node entropy / directionality / motifs are absent from the grammar audit and unrecoverable from its top-20-truncated `function_bigrams` dict) justifies a sibling capability over a fold-in, mirroring the spec-24/31 reuse-not-merge precedent. M1 = **stdlib + numpy** (numpy is an ambient CI transitive dep via scipy); **no networkx** (deliberately avoided to preserve the Tier-1 stdlib contract — every descriptor is computed directly on a numpy adjacency matrix), no model, no GPU, runs unconditionally in CI. Clean-room of the FEATURE CONSTRUCTION from **Authorship Attribution Using Word Network Features** (Segarra, Eisen, Ribeiro; [arXiv:1406.4469](https://arxiv.org/abs/1406.4469)); the paper's attribution **classifier** is explicitly **not** reproduced (that would be a verdict).

Posture: **no verdict** — not authorship/AI, not a quality/readability score, and **not length-controlled** (density and centrality concentration covary with text length and function-word-set coverage; `n_active_nodes` and `total_transitions` are co-reported so the confound is visible). The descriptive structure-concentration band (`diffuse` / `typical` / `concentrated structure`) is carried by NAMED provisional `flagged_signals` + a `variance_audit`-shape `calibration_status` (shipping `n_calibrated: 0`, all-provisional) — there is **no bare `band.score`** (the single most thresholdable artifact; removed per review P2, mirroring the variance-audit discipline of never shipping a bare trust scalar). The edge-total tie pins `total_transitions` to the **run-segmentation primitive** (the same `_tokens_lower` + `len(run) >= 2` content-word-delimited runs), NOT the grammar audit's `.most_common(20)`-truncated `function_bigrams` field — a regression test proves the truncated field diverges on a >20-distinct-bigram fixture (review P1). Below a concrete `total_transitions` floor the band is suppressed (`label: "insufficient structure"`, `band_offered: false`), mirroring `variance_audit`'s "Insufficient signal" band; the raw graph values are still emitted (review P3). A recursive `results`-walk posture guard forbids verdict/selection keys by exact name (and asserts `label` appears only as `band.label`), a hardened networkx guard snapshots `sys.modules` before/after the FWAN import (robust to networkx being importable in the env) plus greps the source and `requirements*.txt`, and an import-disjointness test keeps this descriptive surface out of every held-out detector path (`voice_distance` / `surface_disagreement_resolver`; anti-Goodhart). Two descriptor-semantics defects from a second-pass review are folded (spec 32 §13 P4): (1) the per-node entropy summaries (`min_entropy_node` / `max_entropy_node` / `per_node_mean_bits`) are now computed over **source** nodes only (out-degree > 0), excluding pure **sinks** — function words that appear only as transition targets, whose all-zero outgoing row would otherwise win `argmin` and surface a node with **no** successor distribution as "the most predictable" one while diluting the mean (and the `low_per_node_entropy_mean` band signal) downward; the sink count is reported as `n_sink_nodes` / `n_source_nodes`, `min/max_entropy_node` are `null` iff every node is a sink, and `-0.0` is normalized to `0.0` on emit; (2) the directionality field `weight_asymmetry_mean` is renamed **`reciprocated_weight_asymmetry_mean`** (it is averaged over reciprocated pairs only, so a fully one-directional — maximally asymmetric — graph would otherwise report `0.0` under a name reading as a global asymmetry) and the directionality story is completed by co-reporting **`one_directional_edge_share`** (= `1 - reciprocity`). Surface stays `voice_coherence` (no new `claim_license_surfaces/*.txt`); the capability fragment + per-id drop-in golden are added (no `==N` count bump). The optional spaCy POS-disambiguated node set is flagged as a separate future M2.
