### Added

**`rank_space_audit` — DetectLLM LRR rank-space surprisal signal (spec 32, M1; existing surface `binoculars_discrimination`).** A near-free DERIVED column off the per-position vocab log-prob distributions a causal LM already materializes in `SurprisalBackend.score_text_with_distributions` — the same forward pass `binoculars_audit.py` v2 uses for cross-perplexity. The new signals are the **per-token log-rank series** and the scalars over it: `lrr` (the DetectLLM log-likelihood / log-rank ratio, `mean(surprisal_nats / log(rank+1))`), `log_rank_mean`, `log_rank_sd`, and `log_rank_acf1` — the rank-space analogues of `surprisal_audit`'s `surprisal_mean / surprisal_sd / surprisal_acf_lag1`. Log-rank at a position is an **argsort** of that position's distribution, so there is **no second forward pass** for LRR. This DEEPENS the Tier-4 surprisal family rather than adding a surface (`TASK_SURFACE = "binoculars_discrimination"`, no new `claim_license_surfaces/` file). The rank math lives in a pure-stdlib, model-free helper `scripts/rank_space_signals.py` (`rank_series_from_distributions` + `aggregate_rank_signals`); `scripts/rank_space_audit.py` is the registered surface. M1 is CI-runnable and model-free: the surface takes an **injectable** `distributions_fn` (the `(surprisal_bits, log_probs_nats, token_ids)` tuple), and the unit tests run over deterministic injected stub distributions — no model, no torch. The model path (`SurprisalBackend.score_text_with_distributions`) is the lazy M2 seam, imported only inside `main`. NPR (the perturbation variant) is the GPU-gated M2 / Tier-2 path and is **not built here**: `npr_rank_score` is a fail-loud `NotImplementedError` stub.

**Sign/direction + rank-0 conventions pinned (the family's shared silent failure mode).** Rank is taken over the vocab vector sorted **descending** (rank 0 = highest log-prob, most-probable token); `numpy.argsort` defaults to ascending, which would silently invert every downstream signal, so the helper uses pure-Python ranking (no numpy) with an explicit `(log-prob desc, vocab-id asc)` tie-break and `test_rank_series_fixture` pins that the most-probable token gets `log_rank = log(0+1) = 0.0`, NOT a large value. The rank-0 → inf convention is also pinned: `lrr_t = surprisal_nats / log(rank+1)` is undefined at rank 0 (`log(1) = 0`), so it is emitted as `math.inf` in the per-token series and those positions are **excluded from the `lrr` mean** (count surfaced as `lrr_excluded_positions`); when every position is rank 0 the mean is `None` (a refusal, not a fabricated 0). No `inf` reaches the envelope — the per-token series stays a helper return value and the aggregate scalars are always finite-or-`None`, so they pass the R4 output-validity (finiteness) gate.

**Posture (no verdict).** Descriptive only: the surface emits VALUES (`lrr` + `log_rank_mean/sd/acf1`) plus a PROVISIONAL `band` over the LRR value's OWN axis (`indeterminate` / `low_lrr` / `high_lrr`) carrying `calibration_status: heuristic` + `calibration_anchor: user-baseline-required`, and a claim-license that refuses any AI/human or thresholded verdict. There is no `is_ai` / `is_human` / `label` / `verdict` / `decision` key anywhere. The band names the MEASURED rank-space property, never the inference target (authorship), and the "more LLM-like" DIRECTION is itself the empirical question (M2) and is **not asserted**. The known **ESL / non-native false-positive mode** is surfaced explicitly (log-rank is higher for lexically diverse or non-native prose that uses unexpected-but-valid word choices, so a human ESL writer looks more "surprising" in rank-space — the signal is not ESL-invariant), along with proxy-scorer dependence (a weaker-than-human generator can invert the polarity). Anti-Goodhart: LRR is a comparison-baseline / heuristic, never a held-out audit / fitness / selection signal; promotion past `heuristic` goes only through `scripts/calibration/` against a labeled corpus disjoint from any development corpus. Registered drop-in: `capabilities.d/rank_space_audit.yaml` (`status: literature_anchored`, `tier: surprisal`) + a per-id `_golden_capabilities/rank_space_audit.json` fragment — no shared-file edit, no `==N` count bump.

**Reference.** Su, Zhuo, Wang, Nakov 2023, *"DetectLLM: Leveraging Log Rank Information for Zero-Shot Detection of Machine-Generated Text"*, arXiv:2306.05540 — https://arxiv.org/abs/2306.05540. The paper's +1.75 / +3.9 AUC lifts (WritingPrompts, Table 3) are a LEAD requiring an empirical reproduction on SETEC corpora (M2), NOT a target or a shipped claim.
