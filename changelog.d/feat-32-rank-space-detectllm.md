The new signals are the **per-token log-rank series** and the scalars over it: `lrr` (the DetectLLM log-likelihood / log-rank ratio, `mean(surprisal_nats / log(rank+1))`), `log_rank_mean`, `log_rank_sd`, and `log_rank_acf1` â€” the rank-space analogues of `surprisal_audit`'s `surprisal_mean / surprisal_sd / surprisal_acf_lag1`. Log-rank at a position is an **argsort** of that position's distribution, so there is **no second forward pass** for LRR. This DEEPENS the Tier-4 surprisal family rather than adding a surface (`TASK_SURFACE = "binoculars_discrimination"`, no new `claim_license_surfaces/` file). The rank math lives in a pure-stdlib, model-free helper `scripts/rank_space_signals.py` (`rank_series_from_distributions` + `aggregate_rank_signals`); `scripts/rank_space_audit.py` is the registered surface. M1 is CI-runnable and model-free: the surface takes an **injectable** `distributions_fn` (the `(surprisal_bits, log_probs_nats, token_ids)` tuple), and the unit tests run over deterministic injected stub distributions â€” no model, no torch. **Sign/direction + rank-0 conventions pinned (the family's shared silent failure mode).** Rank is taken over the vocab vector sorted **descending** (rank 0 = highest log-prob, most-probable token); `numpy.argsort` defaults to ascending, which would silently invert every downstream signal, so the helper uses pure-Python ranking (no numpy) with an explicit `(log-prob desc, vocab-id asc)` tie-break and `test_rank_series_fixture` pins that the most-probable token gets `log_rank = log(0+1) = 0.0`, NOT a large value. The rank-0 â†’ inf convention is also pinned: `lrr_t = surprisal_nats / log(rank+1)` is undefined at rank 0 (`log(1) = 0`), so it is emitted as `math.inf` in the per-token series and those positions are **excluded from the `lrr` mean** (count surfaced as `lrr_excluded_positions`); when every position is rank 0 the mean is `None` (a refusal, not a fabricated 0). No `inf` reaches the envelope â€” the per-token series stays a helper return value and the aggregate scalars are always finite-or-`None`, so they pass the R4 output-validity (finiteness) gate.

**Posture (no verdict, NO shipped band).** Descriptive only: the surface emits VALUES (`lrr` + `log_rank_mean/sd/acf1`) and **ships no verdict band** â€” per spec Â§3.5 ("ships no verdict band") / Â§9 ("no threshold is shipped without calibration"), no framework-calibrated LRR cutoff exists and none is invented. By default `band` is `"uncalibrated"` with `thresholds: None` (the `fast_detect_curvature` model): the raw scalars only. A band appears **only** when the operator supplies BOTH `--threshold-low` and `--threshold-high` (their own calibrated operating point), and it then carries `calibration_status: heuristic`, `calibration_anchor: user-baseline-required`, and the `thresholds_operator_supplied_not_framework_calibrated` caveat (the `binoculars_audit` model). There is no `is_ai` / `is_human` / `label` / `verdict` / `decision` key anywhere, and no shipped numeric cutoff. The band names the MEASURED rank-space property, never the inference target (authorship), and the "more LLM-like" DIRECTION is itself the empirical question (M2) and is **not asserted**. The known **ESL / non-native false-positive mode** is surfaced explicitly (log-rank is higher for lexically diverse or non-native prose that uses unexpected-but-valid word choices, so a human ESL writer looks more "surprising" in rank-space â€” the signal is not ESL-invariant), along with proxy-scorer dependence (a weaker-than-human generator can invert the polarity). Anti-Goodhart: LRR is a comparison-baseline / heuristic, never a held-out audit / fitness / selection signal; promotion to a framework-calibrated operating point goes only through `scripts/calibration/` against a labeled corpus disjoint from any development corpus. **Length ceiling (single scoring window).** The rank series indexes `token_ids[t+1]` over a single contiguous sequence, so the target must fit the scorer's context window (~1024 tokens / ~750 words for gpt2). A longer target is chunked by `SurprisalBackend.score_text_with_distributions` â€” each chunk forfeits its first prediction, so the flattened distributions are shorter than `token_ids` and the single-sequence alignment breaks. The surface detects this surplus and **refuses with an actionable `text_too_long` message naming the scorer context window** (not the scorer-blaming "rank computation failed" string, and never a silent mis-rank), warns near the ceiling before scoring, and the capability `do_not_use_when` records the cap so `all_fiction` / `all_nonfiction` is not over-claimed.
