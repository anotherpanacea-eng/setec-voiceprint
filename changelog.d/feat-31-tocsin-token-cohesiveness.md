### Added

**`tocsin_audit` — TOCSIN token-cohesiveness detection signal (spec 31, M1; new surface `token_cohesiveness`).** A black-box, detector-flavored discrimination signal: how stable a text's token set is under repeated seeded random token deletion. `tocsin_audit.py` draws `n_perturbations` random-deletion perturbations of the target, measures each one's semantic difference from the original via an **injectable** `semantic_diff` callable, and reports `token_cohesiveness = 1 - mean(semantic_diff)` in `[0,1]` with its dispersion. M1 ships a pure-stdlib, model-free, CI-runnable default: the `semantic_diff` proxy is a token-set Jaccard distance (`1 - jaccard`, no `numpy`), and the perturbation engine is a single seeded `random.Random` (default seed 1729) over `stylometry_core.word_tokens`. The semantic-difference step is the only model dependency, so it is the M1/M2 seam: M2 swaps in `1 - cosine(embed(original), embed(perturbed))` via `embedding_backend.py` behind the **same** injected callable, with the envelope shape unchanged (only `semantic_diff_backend`, `deletion_unit`, and the numeric values differ). In the literature LLM-generated text exhibits *higher* token cohesiveness than human text, an axis the paper reports as orthogonal to surprisal / curvature — so this is discrimination **evidence**, a member of the multi-signal evidence pack alongside `surprisal_audit` / `fast_detect_curvature` / `intrinsic_dimension_audit`.

**Posture (no verdict).** Descriptive only: the surface emits VALUES (`token_cohesiveness` + SD + raw mean semantic difference) plus a PROVISIONAL `band` over the value's OWN axis (`indeterminate` / `low_cohesiveness` / `high_cohesiveness`) carrying `calibration_status: heuristic` + `calibration_anchor: user-baseline-required`, and a claim-license that refuses any AI/human or thresholded verdict. There is no `is_ai` / `is_human` / `label` / `prediction` / `classification` / `verdict` / `decision` key anywhere in the envelope (asserted by a recursive-walk test). The band names the MEASURED property (cohesiveness), never the inference target (authorship). The capability manifest carries `status: literature_anchored` (the *signal direction* is anchored in the paper) while the *threshold* stays `heuristic` — two distinct objects, asserted separately so the conflation can't regress. Anti-Goodhart: the band thresholds are disjoint from any held-out validation corpus; promotion past `heuristic` goes only through `scripts/calibration/`, never by tuning on the held-out set. Registered drop-in: `capabilities.d/tocsin_audit.yaml` (`tier: core`, stdlib; `dependencies.python: []`, the embedding deps under `python_optional`), `claim_license_surfaces/token_cohesiveness.txt` (the new surface; `VALID_TASK_SURFACES` derives from it), and a per-id `_golden_capabilities/tocsin_audit.json` fragment — no shared-file edit, no `==N` count bump.

**Review hardening (folded).** Three posture/bound fixes on top of the M1 build: (1) the injected `semantic_diff` is **clamped into `[0,1]` at the computing surface** (`min(1, max(0, raw))`) before reduction, so the advertised `token_cohesiveness ∈ [0,1]` invariant is delivered on the load-bearing M2 seam (`1 - cosine` can reach `[0,2]`), not only on the M1 Jaccard default — NaN/inf is *not* clamped so it still reaches the R4 finiteness gate; (2) a **degenerate-perturbation guard**: when `floor(deletion_fraction * target_tokens) == 0` (any text under `1 / deletion_fraction` word tokens — 10 at the default) no token is actually deleted, so the band reports `indeterminate` + a `no_deletion_degenerate` flag instead of reading the vacuous `1.0` ceiling as `high_cohesiveness` (the paper's "more-LLM-like" direction) on a non-event; a new `results.effective_deletions` records the count actually dropped; (3) the **M1 set-level Jaccard proxy saturates near 1.0 on real prose** (set distance bounded by `~deletion_fraction`, far smaller once TTR < 1), so the default M1 band reaches only `high_cohesiveness` — this is now disclosed in the claim-license, `results.assumptions.m1_saturation`, and `band.m1_band_reachability`, and pinned by a test over the real (un-stubbed) Jaccard path. The M1 per-input signal lives mostly in the dispersion (`cohesiveness_sd`); the lower band arms and a wider-moving scalar require a larger `deletion_fraction` or the M2 embedding backend.

**Reference.** Wang, Cheng, et al. 2024, *"Zero-Shot Detection of LLM-Generated Text using Token Cohesiveness"* (TOCSIN), arXiv:2409.16914 — https://arxiv.org/abs/2409.16914.
