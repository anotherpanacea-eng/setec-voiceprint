### Changed

**`discourse_move_signature` — explicit-connective PDTB relation layer (arXiv:2307.03378; `smoothing_diagnosis` surface).** The typed discourse-marker audit gains an **additive** `relation_distribution` block: the writer's **explicit** PDTB top-level discourse-relation mix (Comparison / Contingency / Expansion / Temporal) as descriptive stylometric shape — per-bucket counts, per-1k densities, fractions, and relation entropy. This is an **explicit-connective proxy** of [*Side-by-side Transformers for Implicit Discourse Relation Classification (PDTB-3)*, arXiv:2307.03378](https://arxiv.org/abs/2307.03378): a static, corpus-independent connective lexicon maps each explicitly-signalled connective to its PDTB **majority top-level class**. Polysemous connectives (`while`, `since`, `as`, …) are flattened to one bucket and their share is surfaced as `ambiguous_connective_fraction` (the honesty/confidence valve); the layer does **not** extract argument spans and does **not** disambiguate per-occurrence.

The relation layer is a **parallel, independent count** over the full text — *not* a re-bucketing of the per-sentence move classifier (a two-connective sentence contributes two relation occurrences but types as one move). The implicit-relation classifier that arXiv:2307.03378 actually trains (the larger, harder PDTB subset) is the **gated model-CPU M2 seam** (lazy import + `--include-implicit` default-off + `importorskip` test gate); M1 ships stdlib-only (`re` / `collections.Counter` / `math`, no new deps) and CI-runnable.

Posture: **no verdict** — every leaf is a VALUE (count / density / fraction / entropy); the distribution ships `calibration_status: "uncalibrated"` with **no band**, and the `ClaimLicense` disclaims AI-provenance, argument-quality, and any claim about implicit relations. Optional `--baseline-dir` adds `relation_density_z_scores` framed as distance from the operator's own corpus (a self-baseline), not a population norm. The connective lexicon is a static linguistic inventory, never fit to a SETEC validation/impostor corpus (anti-Goodhart). Tests extend `test_discourse_move_signature.py` + `test_discourse_move_signature_schema.py` (functional pins + no-verdict recursive walk + corpus-independence guard); the pre-existing marker-layer assertions stay green.
