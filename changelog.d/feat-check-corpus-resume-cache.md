### Added

**`check_corpus` is now recoverable at corpus scale (`--records-cache` + resume).** The single-process hygiene loop previously rescored from file 1 on any crash/host hang (the #133 belt/suspenders/buttons gap for this surface). It now accepts `--records-cache PATH`: scored per-file records are flushed atomically (tmp + `os.replace`) every `--records-cache-flush-every` files (default 200) with a `status: in_progress` → `complete` marker, progress is logged to stderr, and a re-run **skips already-scored files** and resumes from the last flush. A compat-meta gate (strip rules/aggressive + warn/fail thresholds) recomputes rather than silently reusing a cache produced under different settings; `--refresh-records-cache` forces a full rescore. Behavior is unchanged when the flag is absent (no cache, no progress — the existing fast path). The sharded `corpus_hygiene` task via `shard_runner` remains the option for RAID-scale; this gives the default single-process path real recovery for mid-size corpora.
