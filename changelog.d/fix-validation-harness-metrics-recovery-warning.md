### Changed

**`validation_harness` now warns when the metrics/bootstrap phase runs un-checkpointed.** The metrics phase is recoverable (per-CI `_MetricsCheckpoint`, atomic, default flush-after-every-CI, resume-from-last-CI, records-fingerprint-gated) **only when `--metrics-cache` is passed**. Without it a SIGTERM / host hang lost the whole bootstrap silently. The harness now emits a loud stderr warning naming the one-flag fix whenever a bootstrap (`--metric-bootstrap-resamples > 0`) runs without `--metrics-cache` — closing the discoverability half of the #133 "belt/suspenders/buttons" gap for this surface. Also refreshes the AGENTS.md worked example + #133 backlog, which still described the metrics phase as honoring "none of the three" (the checkpoint shipped after that note). No change to the cached path or results; whether to make the cache default-on (vs opt-in, matching the scoring cache) is left as an open question on #133.
