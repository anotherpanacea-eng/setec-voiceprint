### Fixed

**`within_doc_segmentation` — Codex P2: zero-MAD profile is not a flat profile (within_doc_segmentation.py).** The round-1 zero-MAD guard gated the entire boundary-detection pass on `MAD == 0`, which suppressed genuine isolated boundaries. MAD is 0 whenever ≥ half the adjacent distances equal the median — true even with a real outlier: `[0,0,0,1,0,0]` has `MAD == 0` and a maximal shift of `1`. Uniform-document false positives were fixed but a sparse genuine register change became a false negative.

- **Flatness is now `max == min`, not `MAD == 0`** (new `_is_flat_profile` helper): only a profile with no within-document variation at all (every distance equal) skips boundary detection. An empty profile is treated as flat.
- **Non-flat zero-MAD fallback** (`analyze_document`): when the profile is not flat but its MAD is 0 — so every MAD-relative threshold has collapsed to the median — boundary detection falls back to a strict-above-median peak rule (`d_i > median`). An isolated spike (e.g. the `1` in `[0,0,0,1,0,0]`) is still emitted as a boundary; a plateau value equal to the median is not.

Regression class `TestSparseSpikeZeroMadRegression` in `tests/test_within_doc_segmentation.py`: a `[0,0,0,1,0,0]`-shaped (zero-MAD, non-flat) profile yields exactly one boundary at the spike seam; a truly uniform (`max == min`) profile still yields zero boundaries; `_is_flat_profile` keys on `max == min`, distinguishing the sparse spike (zero MAD, not flat) from a uniform plateau. The round-1 `TestZeroMadAndZeroNormRegression` uniform-document tests were corrected: the repeated-sentence text is not truly flat (the sliding window's tail seam carries a genuine ~0.996 edge spike), so the invariant asserted is now "no distance-0 plateau window becomes a boundary," and the all-zero-profile case feeds a genuinely flat profile.
