### Added

**`voice_verifier` — LLM-as-verifier authorship surface: one ADVISORY stylometric-consistency signal over a query/reference pair, never a verdict (new surface, spec 31 M1).** A model-based SECOND OPINION that sits beside the stylometric `general_imposters` harness, not a replacement: a pluggable LLM judge (`build_verifier`: mock / manifest / anthropic / openai / gemini) reads two texts as prose and emits a 5-band ordinal `band` — `consistent` / `leans_consistent` / `cannot_determine` / `leans_inconsistent` / `inconsistent` — plus a CAVE-style decomposed `feature_judgements` per a fixed `RATIONALE_FEATURES` register (lexical habits, syntactic constructions, punctuation cadence, discourse moves, register & tone), each a band + note + verbatim span pointers into the named text. **Descriptive, no verdict:** the band vocabulary contains no `same_author` / `different_author` / `ai` / `human` / `forgery` / `plagiar*` token (a blocklist guard asserts it), there is no `p_same_author` / `confidence` / `score` / `verdict` numeric (a shape guard asserts `to_dict()` exposes none — even the M2 model confidence used to demote to `cannot_determine` is consumed internally), and the result ships `calibration_status: "uncalibrated"`, bound to a named judge model + `prompt_fingerprint_sha256`. `cannot_determine` is the designed gray-zone outcome (the `general_imposters` refusal analogue), never an error. **M1 is model-free stdlib:** the `mock` (deterministic, one synthetic grounded span per side) and `manifest` (operator-supplied pre-computed result) judges drive the whole band/rationale/validation/envelope/CLI path with no SDK import; `validate_result` nulls out-of-vocabulary bands to `cannot_determine` with a warning (never silent coercion), drops any span whose offsets/quote do not index its named side (no laundered hallucination), and appends a `rationale_band_mismatch` warning when the per-feature decomposition contradicts the top-level band (the CAVE consistency check) — surfaced, never auto-corrected. Authorship verification is **pairwise**: `--query` and `--reference` are both required (no single-document "is this AI" mode). The `setec run voice_verifier --json` entrypoint emits the schema-1.0 envelope to stdout (`json_delivery: stdout`); a judge failure (incl. a manifest missing the `band` key, a non-JSON body, or a missing SDK) routes through `VerifierError` to a live `available: false` + `reason_category` refusal, not a fabricated band. **Anti-Goodhart / no-selection:** `import voice_verifier` stays stdlib (the SDKs are lazy); the surface ships no new voicewright `SignalSpec` and is never a held-out validator, a selection / `SetecFitness` signal, a reward, or a training target — it is read-only advisory context the human weighs beside the stylometric harness. M2 (gated, `skipif` in CI) wires the real InstructAV/CAVE extraction behind the API adapters + a lazy local backend. Rooted in: Huang et al. 2024, "Can Large Language Models Identify Authorship?" ([arXiv:2403.08213](https://arxiv.org/abs/2403.08213)); Hung et al. 2024, "InstructAV" ([arXiv:2407.12882](https://arxiv.org/abs/2407.12882)); Hao et al. 2024, "CAVE: Controllable Authorship Verification Explanations" ([arXiv:2406.16672](https://arxiv.org/abs/2406.16672)). Implements `specs/31-llm-verifier-authorship.md` (M1).
