### Added

**Eval-discipline / anti-Goodhart protocol upgrades (spec 28, M1) — four model-free hardenings of the validation spine + an over-claim guardrail. NON-capability: no new surface, no new capability id, no new verdict, no golden added.** The thesis: *wire eval discipline before trusting any new detector's reported AUC.* Each item routes to the operator reading a report — never an automated gate, a selection target, or a single trust-me aggregate — and on a detected confound the spine **refuses the pooled headline** rather than emitting a corrected number.

- **Topic-leakage splits (`validation_harness.py`).** A new operator-declared `topic` manifest field (open-set, parsed never inferred; distinct from the existing `topic_match`), `topic_disjoint_split` (HITS group-disjoint partition), and `topic_leakage_diagnostic` (pooled ROC AUC vs. the within-test-topic macro AUC, with the AUC **gap** + a bootstrap CI). The new `--topic-split` flag turns it on; without it the report is byte-for-byte unchanged, but if topic labels are present the harness adds **one** warning pointing at the flag. Roots: Topic Confusion Task (arXiv:2104.08530) and Addressing Topic Leakage / HITS (arXiv:2407.19164).
- **Conformal FPR-bound mode (`conformal_gate.py`).** `threshold_at_fpr_bound` + `--fpr-bound q` emit the conformal threshold whose **reference-class** false-positive rate is bounded by `q` under exchangeability (the operator applies the threshold). One-tailed directions only; `--score` becomes conditionally optional in this mode (still required for the gate). The threshold is a reference-class FPR **ceiling**, NOT P(AI) and NOT a guarantee on the positive (AI) class — the claim-license addendum names that refusal. Root: Multiscaled Conformal Prediction (arXiv:2505.05084); v1 ships the single-scale ceiling (multiscale/Mondrian is a named follow-on).
- **Simpson inversion check (`validation_harness.py`).** `simpson_inversion_check` + `--simpson-check FIELD` (register / topic / length_bucket) detect when the pooled ROC AUC ranking direction contradicts the within-stratum majority and set `pooled_ranking_refused: True` with a refusal message — **detect-and-refuse**, the ESL `refuse_aggregate_fpr` shape; **no corrected aggregate** is emitted. Root: Log-Likelihood & Simpson's Paradox (arXiv:2605.06294).
- **Short-PHD stabilization (`intrinsic_dimension_audit.py`).** `estimate_phd_short` + `audit(short_text_mode="auto")` route short clouds (`n_points < 200`) to a bootstrap-aggregated PHD (median + an inter-fit IQR in `fit.phd_stability`), an honest interval instead of one fragile point. `"auto"` keeps long-text output **bit-for-bit unchanged**; the surface stays uncalibrated — no band, no threshold, no verdict. Root: Short-PHD (arXiv:2504.02873).
- **Over-claim-separability guardrail (DOCS + structural test).** `references/POSTURE_no_overclaim_separability.md` states what the validation spine and `adversarial_robustness_card` will not claim, and `test_no_overclaim_separability.py` makes the absence checkable: the rendered output of `validation_harness`, `conformal_gate`, `intrinsic_dimension_audit`, and `adversarial_robustness_card` carries **no verdict phrase** (a closed list — `this text is ai`, `reliably detects ai`, `reliably separates ai from human`, `ai-detectable`, …; **not** the bare words `reliable`/`separable`, which appear in benign caveats) outside an explicit refusal context. Root: Sadasivan et al. 2023, "Can AI-Generated Text be Reliably Detected?" (arXiv:2303.11156).

Every new mode/flag is **default-preserving** (`--topic-split`, `--fpr-bound`, `--simpson-check`, `short_text_mode="auto"` all leave the un-invoked report byte-for-byte unchanged). The two `capabilities.d/` fragment-text edits (`conformal_gate.yaml`, `intrinsic_dimension_audit.yaml`) are re-blessed in their per-id goldens with **no count bump and no new fragment** (a diff that adds a fragment would have smuggled in a surface). No `voicewright`-consumed contract changes: this spec adds nothing the held-out downstream pulls, so no `CONSUMED_SURFACES` / drift-gate / lock change.

Capability ids touched (fragment-text only, no new id): `conformal_gate`, `intrinsic_dimension_audit`.
