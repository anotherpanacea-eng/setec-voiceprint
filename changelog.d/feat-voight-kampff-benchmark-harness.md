### Added

**Voight-Kampff benchmark harness (`scripts/calibration/`) — held-out external validation of voiceprint's discrimination detectors against the PAN@CLEF Voight-Kampff Subtask-1 labels with the official PAN metric suite.** A three-module eval harness (NOT a task surface, NOT a `capabilities.d/` capability, ships no `_golden_capabilities/` fragment and no `claim_license_surfaces/` file) that runs voiceprint's existing discrimination detectors over the PAN binary human-vs-AI dataset (where the LLM was instructed to **mimic a specific human author**) and scores them with the official PAN metrics:

- `pan_voight_kampff_to_manifest.py` — adapter from a locally-staged PAN release (JSONL **and** CSV, BOM-tolerant, streaming) to a SETEC manifest slice (`use: ["validation"]` as a **list**, `privacy: "shareable"`; human → `pre_ai_human`, machine → `ai_generated`). Vendors **no** PAN data — writes a `NOTICE.md` next to the spilled text; the dataset **download is an out-of-M1 seam** and the PAN field-map is documented as the single edit point.
- `pan_metrics.py` — the PAN metric suite (`roc_auc`, `brier`, `c@1`, `f1`, `f0.5u`, `pan_mean`) re-implemented in **stdlib**, anchored **verbatim** to the Apache-2.0 TIRA evaluator (`pan-webis-de/pan25-generative-ai-authorship-verification`). `roc_auc` reuses `validation_harness.fallback_roc_auc`; per-metric CIs reuse `validation_harness.paired_bootstrap_ci`. The non-response convention (prediction `== 0.5`) drives c@1 and F0.5u exactly per the reference — verified bit-exact against sklearn across cases including abstentions and one-class gold.
- `pan_voight_kampff_benchmark.py` — runner + report assembler. The runner import-and-calls each detector (binoculars via its injected `score_fn`, plus a stdlib stand-in) so the whole adapter→runner→scorer→report pipeline runs on CPU with **zero model loads**; orientation is declared per detector (binoculars' `lower_is_ai` sign-flipped to higher-is-AI). The detector's **existing two-threshold band** is wired as the label-free abstention zone (the `indeterminate`/uncalibrated band → PAN's 0.5 non-response).

**Anti-Goodhart (load-bearing, structural):** the benchmark is external held-out validation, never a tuning/calibration/selection target. The harness writes **only** a report; it imports no fitter; PAN labels flow one-way (PAN → adapter → runner → scorer → report, terminal); the Brier probability transform is declared, fixed, and label-free (min-max over the run, never fitted to a PAN metric); and thresholded cells (c@1/F1/F0.5u) are **null** with reason `no_operating_point_without_fitting_to_pan` unless the operator supplies an operating point — the harness never sweeps a threshold against the PAN labels (PAN's own `--optimize-score` sweep is deliberately not re-implemented).

Binoculars is one of the three official PAN baselines and ships first-party (recomputed), so the `binoculars` baseline row is `source: "first_party"`; TF-IDF+SVM and PPMd are cited PAN-published numbers in M1 (recomputation is a named seam).

Cites: PAN@CLEF 2025 Generative AI Authorship Verification, Subtask 1 (Voight-Kampff) — task page <https://pan.webis.de/clef25/pan25-web/generated-content-analysis.html>; dataset **Zenodo record 14962653** (reused by the 2026 edition); the Apache-2.0 TIRA evaluator <https://github.com/pan-webis-de/pan25-generative-ai-authorship-verification>; and the Binoculars baseline (Hans et al. 2024, [arXiv:2401.12070](https://arxiv.org/abs/2401.12070)).
