### Added

**`embedding_attribution` — a faithful glass-box ATTRIBUTION of the LUAR cosine (new `embedding_attribution` surface).** The HIATUS glass-box explanation layer on the `authorship_embedding` / `voice_fingerprint` LUAR cosine. Where `cosine_explanation` (spec 27) placed the opaque cosine *side by side* with SETEC's named interpretable stylometric features (burstiness, MATTR, MTLD, function-word ratio, dependency distance) and flagged `tracks`/`diverges`, this surface ATTRIBUTES the cosine: it reports, per named feature, a **signed contribution** to the modeled similarity (Latent-Space Interpretation, [arXiv:2409.07072](https://arxiv.org/abs/2409.07072)) and decomposes the cosine into an interpretable-feature-**explained** part and a **faithful residual** — the slice the named features genuinely do not capture (Residualized Similarity, [arXiv:2510.05362](https://arxiv.org/abs/2510.05362)). Program lineage: IARPA **HIATUS** (Human Interpretable Attribution of Text Using Underlying Structure) / the **AUTHOR** consortium; both source papers are HIATUS work, and the fleet already vendors LUAR (Rivera-Soto et al., EMNLP 2021) as the `voice_fingerprint` encoder, so this layer attaches to the same frozen manifold.

**Explanation, not verdict.** DESCRIPTIVE only — it explains `voice_fingerprint` and never replaces its posture. It inherits and re-states `authorship_embedding`'s refusals (no same-author / different-author / AI-vs-human) and adds none, pinned by a **recursive** no-verdict walk over the whole `results` tree. The residual is an explanation-**coverage** quantity (the part the named lens does not capture), never authenticity / suspicion / AI-ness, and is never compared to a threshold; a negative signed contribution means the named feature reads as *divergent* for the pair, not "less likely same author". `coverage_band ∈ {well-named, mostly-named, largely-unnamed, indeterminate}` names the coverage of the explanation, never an inference target.

**The explained_fraction, disciplined (operator decision; consciously extends spec 27 Open-Q1).** Spec 27 withheld the `explained_fraction` because you cannot partition a neural cosine without a fit. This surface **ships** the fitted explained/residual split (the fit is the whole point of arXiv:2510.05362) but reconciles it with spec 27's caution: the **default/headline** view is the spec-27 side-by-side + per-feature agreement, and the numeric `explained_fraction` is carried but surfaced as **calibrated only when `calibration_status` warrants it** (it ships `uncalibrated` → `fraction_calibrated: false`). The fit is a **declared input** with explicit provenance (`fit_source ∈ {injected_model, corpus_fit, unfit}`, `model_id`, corpus `r2` + `n_corpus_rows`), never an invented per-pair operating point. An **unfit** model or a **near-zero-cosine** single-pair degeneracy ABSTAINS (`coverage_band: indeterminate`, no fraction) rather than fabricating a split.

**Anti-Goodhart held-out disjointness.** The attribution basis is EXACTLY `cosine_explanation`'s curated named-feature set (the interpretable-scalar axis), never re-derived from the LUAR embedding itself — the residual *is* the measured non-overlap between the named axis and the learned manifold, which is what keeps the two axes independent. It computes no new authorship number a downstream loop could Goodhart toward; it decomposes the existing one.

**M1 / M2 split.** M1 (shipped, fully tested) runs the attribution + decomposition over INJECTED embeddings / named-feature vectors — pure Python + numpy, CI-runnable, no torch / transformers / network / model weights. The live-LUAR path (`compute_inputs`) is a lazy-import + `skipif(no torch)` M2 seam, exactly like `cosine_explanation.compute_inputs` and `voice_fingerprint._load_encoder`; absent style-embedding tier → a clean `missing_dependency` envelope, not a traceback. The learned latent-direction anchors of arXiv:2409.07072 (LUAR-embedding a labeled attribute corpus) are POC-gated M2. A precomputed `--inputs-json` path carries `inputs_source: "injected"`, consumes no text, rides no privacy gate, and the claim license refuses reading it as a production interpretation. Inputs are fail-loud: no `--comparison`/`--inputs-json`, empty/malformed features, a non-finite injected cosine, or a malformed `attribution_model` are all clean `bad_input`. Ships `literature_anchored` (close to the two published conditions; the operating point for your corpus stays uncalibrated — promotion is gated on the HIATUS HRS study). Implements the `explainable-embedding-attribution` spec (M1).
