### Added

**`model_family_attribution` — abstention-first, raw per-family similarity ranking (spec 32, M1; new `model_family_attribution` surface).** A new advisory axis answering "which supplied model family does this prose read most like?" — as a **raw, un-normalized, abstention-gated ranking**, never a verdict. Ranks a `--target` against operator-supplied per-family reference corpora (a JSONL `--reference-manifest` of `{family, text|text_path}` or a `--reference-dir` of `<family>/<files>` subdirs, via a new `_load_family_dir` subdir-grouping loader) over a small set of standardized named stylometric features reused model-free from `variance_audit` (burstiness_B / MATTR / MTLD / function-word ratio / mean-dependency-distance, the last spaCy-gated). Pure stdlib, deterministic, no embedding model — the LUAR/style-embedding backend is M2 (gated), and M1 stands alone. Self-excluding (the target is dropped from its own family corpus, mirroring `general_imposters`).

Posture — **no verdict, load-bearing.** M1 emits ONLY raw per-family `similarity` (un-normalized, in (0, 1]) + a ranking; **nothing sums to 1 over families** (a normalized posterior manufactures a P(family) reading that looks confident even when every family fits badly). Features are robust-z standardized (median + 1.4826·MAD over the pooled reference) before aggregation, so a large-scale feature (MTLD ∈ ~[10, 200+]) cannot dominate, and the comparison subspace is resolved ONCE (intersection across target + all references) so the spaCy-gated `mdd` is uniformly in or out, never per-doc. Abstention is REAL and the default: `attribution_available` is True only when every gate passes — `< 2` reference families, any family below the 5-doc floor, a **relative** out-of-distribution target (distance to the top family exceeds `--ood-k`× that family's *own within-scatter*, not a fixed floor, so it can tell "true source absent" from "register mismatch"), a top-2 margin below `--margin`, or a `human` label that would occupy the top slot (the AI/human axis belongs to the discrimination surfaces, which also refuse it). The `family_ranking` is still emitted as raw evidence when not attributable, flagged. The claim license refuses any "produced by `<family>`" attribution, any AI-vs-human ruling, any calibrated probability / normalized posterior, and naming a family absent from the references; ships `uncalibrated` with PROVISIONAL thresholds. Ships with stdlib tests covering the deterministic ranking, the no-key-sums-to-1 / no-verdict-key guards, each real abstention gate, standardization dominance, the fixed-subspace (mdd in/out) invariant, robust `bad_input` handling, and self-exclusion. Clean-room; cites **arXiv:2410.16107** (Biber-feature separation of LLM families — ~96 features over large corpora, vs M1's ~5 over operator corpora, so M1 is deliberately weak), **arXiv:2309.13322** (From Text to Source — the attribution task), and **arXiv:2504.11369** (OpenTuringBench — the open-set / OOD framing the relative gate addresses).
