### Fixed

**`imessage_sent_atomic` now has a native Windows handle-relative portable-tree backend.** Offline approved bootstrap, resumable snapshot copying, row publication, compare-and-swap state updates, cleanup, and sealing resolve single path components beneath pinned NT directory handles, reject reparse points and hard-linked mutable files, and preserve fail-closed destination guards. Transient Windows sharing and access failures during sealed directory publication receive bounded retries at 100, 300, and 900 milliseconds with binding and content revalidation; exhausted retries leave the destination untouched. A restarted run completes only a schema-valid, hash-chained row-journal rewrite interrupted at its temporary or predecessor name. Stable empty SQLite WAL and SHM sidecars are tolerated with runtime warnings, while nonempty or indirected sidecars still refuse. A focused Windows CI job covers the writer and immutable-root guard contract.
