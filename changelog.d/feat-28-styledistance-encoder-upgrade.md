### Added

**`voice_fingerprint` / `crosslingual_voice_distance` — StyleDistance + mUAR style encoders behind the existing `_load_encoder` seam (spec 28, M1).** Two stronger, more content-independent style encoders are now selectable behind the *existing* injectable seam — an **encoder upgrade, not a new surface**: no new capability `id`, no new aggregate, no new band, no threshold, and the cosine-distribution contract is untouched. `voice_fingerprint.MODEL_ALIASES` gains `styledistance` (synthetic near-paraphrase contrastive training for a *more content-independent* manifold than LUAR — [arXiv:2410.12757](https://arxiv.org/abs/2410.12757)) and `muar` (Multilingual Universal Authorship Representation — [arXiv:2509.16531](https://arxiv.org/abs/2509.16531)), each routed through the reused `transformers` dispatch + dependency gate via a lazy `_StyleDistanceEncoder` / `_MUAREncoder` class (present in code, **never executed under the unit suite** — the spec-02 `_LUAREncoder` discipline; the real weight load is the `skipif`-gated M2 model seam). `crosslingual_voice_distance` gains an **opt-in `--encoder muar`** flag (default **off**) that reports a learned mUAR cosine block *beside* the parser-free `delta` (never replacing it), with the `voice_fingerprint` reuse imported **lazily and in-branch** so the default invocation stays import-time stdlib / zero-dependency (`dependencies.python: []`).

**Posture (descriptive-only, never a verdict; held-out firewall preserved).** `DEFAULT_MODEL` stays `luar` — no silent default flip — and every encoder ships **PROVISIONAL** (an encoder swap does not promote `calibration_status`). `_claim_license`'s formerly-static caveat block is **refactored into a per-encoder branch** (LUAR register skew / Wegmann punctuation-casing / StyleDistance "more content-controlled, not topic-proof" / mUAR "multilingual representation does not license cross-language comparison"), and the cross-model-incomparability caveat now enumerates all four — while the `licenses` and `does_not_license` **refusal strings ("SAME PERSON" / "DIFFERENT AUTHOR" / "AI-generated or human-written") are byte-for-byte unchanged across every encoder** (tested). The new encoder module/aliases are **not imported by** the held-out validators `general_imposters` / `mimicry_cosplay_audit` / `binoculars_audit` (structural import-disjointness test) — import-disjointness is *necessary, not sufficient*: it closes the shared-code leak; the correlation leak (selection encoder id ≠ any held-out backend id) is the consumer drift gate's enforcement, named in spec 28. The `--lang` same-language refusal is unchanged even in `--encoder muar` mode (multilingual is a *capability*, not a *license*). Implements `specs/28-styledistance-encoder-upgrade.md` (M1).
