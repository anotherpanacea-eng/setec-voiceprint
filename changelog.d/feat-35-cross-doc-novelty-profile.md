### Added

**`cross_doc_novelty_profile` — per-feature cross-document novelty PROFILE (spec wave-4, M1; new id on `set_level_diversity`).** A new *set-level* axis that answers: for ONE target document vs a supplied reference POOL, *which named stylometric features make the target atypical*, and by how far? For each of the 7 stdlib feature families (`function_words` / `punctuation` / `paragraph_dialogue` / `pronoun_modal_negation` / `char_ngrams_3/4/5` — the `extract_features(include_spacy=False)` set), reports a per-named-feature z-position (mean/SD z) of the target relative to the pool's distribution, plus a per-family |z| distribution summary (7-key n/mean/sd/min/p10/p50/p90 block). The per-feature distribution + position table ARE the read; the pool's coordinate system is fixed by `select_feature_names(pool)` so the target is placed *into the pool's axes* (target-only features reported separately in `target_only_features`, never z-scored). M1 is **model-free** (stdlib z-position arithmetic, forced `include_spacy=False`, no torch/spaCy, CI-runnable). Self-exclusion mirrors `corpus_novelty_audit`: a pool entry whose resolved path equals the target's is dropped before the distribution is built.

Clean-room of the per-feature-vs-population position idea from **GENIE: Generative Note Information Extraction** ([arXiv:2606.12790](https://arxiv.org/abs/2606.12790)) — which decomposes novelty into named task-specific features — combined with the no-single-scalar / distribution-is-the-read posture from **NoveltyBench: Evaluating Language Models for Humanlike Diversity** ([arXiv:2504.05228](https://arxiv.org/abs/2504.05228)).

This id is the **feature-wise complement** to the cluster-wise `distinct_diversity_audit` (that one partitions a SET into near-dup equivalence classes; this one asks which named stylometric features make ONE target atypical vs a pool). Neither subsumes the other.

Posture: **no verdict.** No single "novelty score", no band, no AI/human/derivative determination. Low novelty is NOT "AI/derivative": a tight or single-source pool, a shared genre/register, a templated form, or a house style all lower apparent novelty legitimately. A per-feature z-position is a measurement, not a provenance call. The no-verdict guard is a recursive `_walk_keys` + `_FORBIDDEN_RESULT_KEYS` CI-blocking unconditional assertion (modeled on `test_distinct_diversity_audit.py`). Ships `status: heuristic`, `calibration_status: provisional`. Robust median/MAD z is an explicitly deferred M2 upgrade. The embedding/authorship lens is a POC-gated M2 seam (`--lens embedding` → `missing_dependency`, fails loud on import-SUCCESS too, never silently falls back to the stylometric lens). Ships with 27 acceptance-criteria tests (AC-1 through AC-20).
