### Added

**`originality_audit` — DJ-Search reconstructibility vs a reference pool (spec 22, M1; new `set_level_diversity` surface).** A new *set-level* axis: every existing surface scores one document against a baseline, whereas `originality_audit` scores a target against a whole *pool* of reference material it might recombine. Greedily covers the `--target` with the longest left-to-right verbatim token spans (>= `--min-ngram`, default 8) found anywhere in a `--reference-dir` / `--manifest` (the operator's impostor pool by default), and reports `coverage` (= reconstructibility) and `originality = 1 - coverage`, oriented gt = *less reconstructible from that pool*. Clean-room reimplementation of **DJ Search** from *AI as Humanity's Salieri: Quantifying Linguistic Creativity* ([arXiv:2410.04265](https://arxiv.org/abs/2410.04265)); pure stdlib, deterministic, no model.

Posture: **no verdict.** Low originality is NOT "AI" and high originality is NOT "human" — a thin/narrow pool inflates it, and quotation / shared sources / genre formula legitimately lower it; the claim-license refuses any AI/human, plagiarism, or copyright determination, and thresholds are operator-side / PROVISIONAL (`status: heuristic`). Self-exclusion drops any reference doc identical to the target path (mirrors `general_imposters`), so a target never reconstructs itself. Ships with 14 stdlib tests. The companion **M2 `homogeneity_audit`** (within-set pairwise-cosine + effective modes) is specced and POC-gated (BRAINSTORM #3 / Artificial Hivemind, [arXiv:2510.22954](https://arxiv.org/abs/2510.22954); QUDsim [arXiv:2504.09373](https://arxiv.org/abs/2504.09373)) — not in this change.
