### Changed

**Deduplicated the R1 field-bundle validator into a shared `tools/r1_bundle.py`.** `validate_r1_bundle` (plus its constants) previously lived as two hand-synced copies — in `tools/check_capabilities_drift.py` (the drift linter) and `tools/seed_capabilities.py` (the seeder) — and they had drifted: only the linter's copy knew the `group` / `required_groups` rules. Both tools now import the single, pure, stdlib-only validator from `tools/r1_bundle.py`, so they can never diverge again. The seeder stays self-contained — it imports the small validator, not the drift linter. No behavior change (the seeder emits no `min_setec_version` bundle today, so its now-stricter validation is dormant); a regression test pins that both tools resolve to the one shared validator object.
