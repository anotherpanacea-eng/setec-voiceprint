### Fixed

**`house_style_decomposition` — fail-CLOSED the must-differ org check on null/blank `org_id` (Codex round-2).** The per-level identity-membership gate's must-differ branch (`org_rule is False`, i.e. `different_context` and `same_genre_outside_org`) only rejected an entry whose `org_id == target_org`; a `None`/blank `org_id` slipped through as "a different house" even though membership is then unprovable, so a null-org entry could pose as a different venue and let house style masquerade as idiolect. The check now requires a **non-blank** `org_id` that is provably **!= target_org** for every must-differ level — a missing/blank org is **refused** (`HouseStyleError` → `bad_input`), matching the fail-closed posture already enforced on the must-equal (`org_rule is True`) levels. The built-in `--check-all` worked-example fixture and the test fixture's `different_context` entries now carry a non-null differing venue org so the happy path still passes. New regressions pin: null/blank `org_id` under `different_context` → `bad_input`; null `org_id` under `same_genre_outside_org` → `bad_input`; a valid differing non-blank org still passes.
