### Added

**`distinct_diversity_audit` — set-level distinct-CLUSTER diversity (spec 33, M1; `set_level_diversity` surface).** A new distinct-output axis on the existing set-level surface: where `homogeneity_audit` reads a *continuous* average-pairwise-cosine spread, this **partitions** a POOL of N responses to one prompt into near-duplicate equivalence clusters and reports the *partition* — the distinct-cluster count, the cluster-size distribution (the 7-key `n/mean/sd/min/p10/p50/p90` block), the explicit sorted `cluster_sizes`, a `distinct_ratio` (= `n_clusters / n_texts`, oriented gt = *more distinct*), a utility-weighted distinctness at an operator-supplied discount, and one **positional** representative per cluster (the earliest member by input order, with `size` + `member_ids` + an excerpt). The read is *how many genuinely distinct things the model said, and what each looks like* — **never a single "diversity score."** Clean-room reimplementation of the set-partition read from **NoveltyBench: Evaluating Language Models for Humanlike Diversity** ([arXiv:2504.05228](https://arxiv.org/abs/2504.05228)); the paper's learned deduper is replaced by a stdlib **lexical near-dup** equivalence relation (word-shingle Jaccard `>=` threshold, single-link transitive closure) so M1 is pure-stdlib, no numpy, deterministic. The model/embedding deduper is the POC-gated M2 `--lens model-dedup` seam (lazy-import + fail-loud `missing_dependency`, not in this change).

Posture: **no verdict.** Low distinctness is NOT a model defect and is NOT "AI" — a tight topical prompt, a shared genre, or a single source collapses clusters with no model involvement; mixing prompts inflates apparent distinctness. **No absolute band** (like `homogeneity_audit` / `originality_audit`); thresholding is operator-side / PROVISIONAL (`status: heuristic`). The representative is **positional** (earliest by input order), never a "best" / ranked / selected output — the surface picks no winner. The recursive no-verdict field guard holds (`results` is disjoint from `{is_ai, is_human, verdict, label, same_author, score}`; no `band` / `diversity_score`). The `distinct_ratio` `(0, 1]` bound is a property of the surface's own arithmetic, not of the R4 gate (a `ratio`-named leaf gets only finiteness); with `--utility-discount > 0` the utility-weighted value can exceed `n_clusters` by design. Differentiated from its surface-mate `skeleton_overlap_audit` (CONTENT near-dup + full partition incl. singletons vs STRUCTURAL discourse-template LCS + singletons dropped). The model-dedup seam fails loud **even when a `noveltybench_deduper` module is importable** (M1 wires no real deduper, so the import-SUCCESS branch returns `missing_dependency` rather than silently falling back to the lexical lens) — closing a planted false-invariant. Ships with 22 stdlib tests.
