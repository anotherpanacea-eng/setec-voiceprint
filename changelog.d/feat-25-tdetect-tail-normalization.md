### Changed

**`fast_detect_curvature` — opt-in T-Detect Student-t tail-aware normalization (spec 25).** Adds `--tail student-t` (default `gaussian`, unchanged) and `--t-df` (default 5, > 2). The Fast-DetectGPT curvature z-score assumes a Gaussian null, which mis-bands adversarial / paraphrased machine text (leptokurtic, heavy-tailed). T-Detect ([arXiv:2507.23577](https://arxiv.org/abs/2507.23577)) standardizes against a Student-t(ν): under `--tail student-t` the `results` additionally carry `p_value_t` — the **deliverable**, the survival under a heavy-tailed t(ν) null, which is *less extreme* than the Gaussian for the same deviation (fewer false positives on adversarial text) — plus `curvature_t` (= `curvature_score / √(ν/(ν−2))`, a constant rescale reported only as `p_value_t`'s by-product), `tail`, and `t_df`. Only the normalization changes; sampling and per-token discrepancy are identical to Fast-DetectGPT.

**Default-preserving:** with no `--tail` (or `--tail gaussian`) the envelope is byte-for-byte unchanged. Posture (no verdict): `p_value_t` is named in `does_not_license` as "a tail-aware significance, NOT a probability the text is AI" (only when emitted, so the gaussian claim license is unchanged); thresholds stay operator-side / PROVISIONAL. 10 stdlib/scipy tests (torch-free via the stub backend); the existing 13 curvature tests pass unchanged.
