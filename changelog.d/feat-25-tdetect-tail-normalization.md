### Changed

**`fast_detect_curvature` — opt-in T-Detect Student-t tail-aware normalization (spec 25).** Adds `--tail student-t` (default `gaussian`, unchanged) and `--t-df` (default 5, > 2). The Fast-DetectGPT curvature z-score assumes a Gaussian null, which mis-bands adversarial / paraphrased machine text (leptokurtic, heavy-tailed). T-Detect ([arXiv:2507.23577](https://arxiv.org/abs/2507.23577)) standardizes against a Student-t(ν): under `--tail student-t` the `results` carry **`curvature_t` — the deliverable**, the t-standardized curvature SCORE (`= curvature_score / √(ν/(ν−2))`) that the paper and its reference implementation expose. Because ν is fixed, `curvature_t` is a global constant rescale of the Gaussian `curvature_score`, so its discrimination *ranking* equals the Gaussian z — T-Detect changes the reference **null**, not the ranking. Also carried: `p_value_t` — a **secondary, UNCALIBRATED** tail probability of `curvature_t` under the assumed t(ν) null (an aid for thresholding, less extreme than the Gaussian for the same deviation; the ν null is a modeling assumption, not a fitted calibration), plus `tail` and `t_df`. Only the normalization changes; sampling and per-token discrepancy are identical to Fast-DetectGPT.

**Default-preserving:** with no `--tail` (or `--tail gaussian`) the envelope is byte-for-byte unchanged. Posture (no verdict): `does_not_license` names `curvature_t` as the deliverable and flags `p_value_t` as "an UNCALIBRATED tail aid … NOT a probability the text is AI" (only when the student-t mode runs, so the gaussian claim license is unchanged); thresholds stay operator-side / PROVISIONAL. The `t_df ≤ 2` case (Student-t variance ν/(ν−2) undefined) is rejected both at the CLI **and** inside `audit()` for direct callers. 11 stdlib/scipy tests (torch-free via the stub backend); the existing 13 curvature tests pass unchanged.
