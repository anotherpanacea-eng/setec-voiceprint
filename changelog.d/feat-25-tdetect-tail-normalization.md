### Changed

**`fast_detect_curvature` — opt-in T-Detect Student-t tail-aware normalization (spec 25).** Adds `--tail student-t` (default `gaussian`, unchanged) and `--t-df` (default 5, > 2). The Fast-DetectGPT curvature z-score assumes a Gaussian null, which mis-bands adversarial / paraphrased machine text (leptokurtic, heavy-tailed). T-Detect ([arXiv:2507.23577](https://arxiv.org/abs/2507.23577)) standardizes against a Student-t(ν): under `--tail student-t` the `results` carry **`curvature_t` — the deliverable**, the t-standardized curvature SCORE (`= curvature_score / √(ν/(ν−2))`) that the paper and its reference implementation expose, plus `tail` and `t_df`. Because ν is fixed, `curvature_t` is a global constant rescale of the Gaussian `curvature_score`, so its discrimination *ranking* equals the Gaussian z — T-Detect changes the reference **scale**, not the ranking. **No p-value is emitted:** `curvature_t` is a rescaled (asymptotically Gaussian) z-score, not a Student-t variate, so a t-survival of it would be an *unsupported* transform, not a calibrated probability. Only the normalization changes; sampling and per-token discrepancy are identical to Fast-DetectGPT.

**Default-preserving:** with no `--tail` (or `--tail gaussian`) the envelope is byte-for-byte unchanged. Posture (no verdict): `does_not_license` names `curvature_t` as the deliverable and states that no p-value is emitted (it would be unsupported); thresholds stay operator-side / PROVISIONAL. The `t_df ≤ 2` case (Student-t variance ν/(ν−2) undefined) **and** an unknown programmatic `tail` value are both rejected at the CLI (`choices=`) **and** inside `audit()` for direct callers (the latter never silently falls through to gaussian). 11 stdlib/scipy tests (torch-free via the stub backend); the existing 13 curvature tests pass unchanged.
