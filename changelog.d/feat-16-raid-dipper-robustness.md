### Added

**RAID adversarial transforms + `paraphrase_ladder.py` — recursive-paraphrase decay-curve harness (spec 16, M1).** Two additive pieces on the existing `validation` surface, both model-free / CPU-only / stdlib:

- **RAID-attack transforms in `adversarial_fixtures.py`** (additive, default-preserving). Nine new deterministic, pure `text -> text` transforms modeling the RAID benchmark's adversarial attacks (Dugan et al. 2024, "RAID: A Shared Benchmark for Robust Evaluation of Machine-Generated Text Detectors", arXiv:2405.07940): `article_deletion`, `number_swap`, `paragraph_shuffle`, `misspelling`, `alternative_spelling`, `insert_paragraph`, `case_swap`, `whitespace`, `synonym_swap` (the synonym/spelling tables are small bundled closed lookups, NOT models). A `RAID_ATTACK_CLASSES` register maps each attack to its transform, and a `--raid-suite` CLI mode emits one `(clean, obfuscated)` pair per RAID attack into the `pairs.jsonl` layout `pan_replay` already consumes (the two compose with no glue). The three existing tokenizer transforms (`zero_width` / `soft_hyphen` / `homoglyph`) and their `TRANSFORMS` keys are unchanged; a `pan_replay` run over a fixture dir of only the original four PAN classes is byte-identical to before. **No `pan_replay` change is required** — it already scores every `obfuscation_class` string present in the manifest (it has no known-class-vocabulary filter), so RAID classes flow through unchanged.

- **`paraphrase_ladder.py`** — a new `validation`-surface tool (capability id `paraphrase_ladder`) that walks a *graded ladder* of paraphrase passes (a text paraphrased again and again) and emits a per-(signal × rung) **decay curve** for SETEC's existing signals. It re-runs the same `audit_text(do_tier4=False)` + `classify_compression` scoring `pan_replay._score_text` uses, builds ONE robustness card (rung 0 = base, rungs 1..N = fixture columns), and extracts only the card's per-CELL `base_value` / `relative_change` / `card_label` (the `pan_replay` cell-extraction pattern) — the card's top-level aggregate dict is never embedded. The bundled M1 paraphraser is a deterministic stdlib proxy (`build_proxy_ladder`, composed from the RAID primitives), honestly labeled `proxy_stdlib` everywhere so a flat proxy curve is never read as a DIPPER-grade result. The realistic recursive paraphrase (DIPPER, arXiv:2303.13408) is the GPU-gated M2 seam, in a later PR.

Posture (the `pan_replay` line, carried forward and hardened): the surface emits the per-(signal × rung) decay list and a descriptive per-signal `monotone: bool`, but **no** aggregate robustness or accuracy score — no `robustness_score` / `auc_retained` / `area_under_decay` / `is_robust` / `n_robust_signals` / `overall_robustness` field anywhere in the serialized payload (the extended `_walk` banned-key test enforces it at every depth). It is never a selector, a calibration-threshold input, or a reward; it re-labels no fixture and mutates no manifest (no module-level function both takes and returns a fixture/manifest object); it imports nothing from `calibrate_thresholds` / `conformal_gate`; `import` stays stdlib (the DIPPER paraphraser is a lazy M2 seam). The `ClaimLicense` **refuses** any detector-accuracy headline AND — load-bearing — any claim that a signal is "robust to paraphrase," quoting Sadasivan's human/AI-overlap separability ceiling verbatim (Sadasivan et al. 2023, "Can AI-Generated Text Be Reliably Detected?", arXiv:2303.11156): a flat decay curve means the attack did not erode S at THIS paraphrase strength, never that S is paraphrase-robust. The license also states that no per-rung Δ is a retention threshold and that ranking robustness by comparing Δ across signals/rungs is not licensed (closing the only residual thresholding affordance). Monotonicity is observed, never enforced — a non-monotone curve scores normally with `monotone: False`. RAID / DIPPER corpora are not vendored (redistribution-gated; the bundled fixtures are tiny synthetic stand-ins).

A `feat` (MINOR): one new capability (`paraphrase_ladder`) on the existing `validation` surface (no new task-surface label), plus additive RAID transforms on `adversarial_fixtures.py`. Roots: RAID (arXiv:2405.07940), DIPPER (arXiv:2303.13408), Sadasivan et al. (arXiv:2303.11156).
