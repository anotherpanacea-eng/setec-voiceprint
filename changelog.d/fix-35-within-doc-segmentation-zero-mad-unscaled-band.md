### Fixed

**`within_doc_segmentation` — Codex round-3 P2: zero-MAD band must be `unscaled`, not `marked_shift` (within_doc_segmentation.py).** The round-2 fix correctly *accepted* a peak strictly above the median on a non-flat zero-MAD profile, but it still passed the original thresholds to `_assign_band`. When the within-document MAD is 0, the `median + {1,k,4}*MAD` ladder collapses (`T_slight == T_moderate == T_marked == median`), so every accepted peak — even a `1e-12` deviation — satisfied `d_i >= T_marked` and was labeled `marked_shift`. That replaced the round-1 false negative with a false STRONGEST-severity label.

- **New band `unscaled`** added to `BAND_VOCAB`: a register discontinuity IS present but its severity cannot be scaled against a zero-dispersion within-document profile. It is a sideways "magnitude not comparable" tag, NOT an ordinal rung above `marked_shift`.
- **Non-flat zero-MAD path** (`analyze_document`): a peak strictly above the median (MAD == 0, but `max != min`) is now assigned `unscaled` directly — the collapsed `median`-equal thresholds are never reused. `[0,0,0,1e-12,0,0]` → `unscaled` (was `marked_shift`); a genuine large zero-MAD spike `[0,0,0,0.9,0,0]` → also `unscaled` (honest: severity unscalable, neither over- nor under-claimed).
- **Unchanged paths:** a truly flat profile (`max == min`) still emits ZERO boundaries; a normal-dispersion profile (MAD > 0) still uses the ordinal `none/slight/moderate/marked` ladder via `_assign_band` (byte-unchanged) and never produces `unscaled`. `_assign_band` is now documented as the MAD > 0 ladder only.
- **Firewall intact:** `unscaled ∈ BAND_VOCAB`, so the `assert_no_authorship` band whitelist (Rule 4) accepts it; it carries no authorship/identity token.

Regression class `TestZeroMadUnscaledBandRegression` in `tests/test_within_doc_segmentation.py`: tiny positive spike `[0,0,0,1e-12,0,0]` → band `unscaled` (pre-fix: `marked_shift`); genuine large zero-MAD spike → `unscaled`; truly uniform → zero boundaries; a normal MAD > 0 profile keeps its ordinal band (and the strong outlier is `marked_shift`, never `unscaled`); and a magnitude sweep pinning that no collapsed-threshold `marked_shift` is reachable for MAD == 0. The `capabilities.d` / `_golden_capabilities` purpose prose, the `assumptions.no_band_absolute` field, and `references/signals-glossary.md` now document `unscaled`.
