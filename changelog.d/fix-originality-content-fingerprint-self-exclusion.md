### Fixed

**`originality_audit` no longer lets the target reconstruct itself from an inline-text copy in the reference pool (path-only self-exclusion hole).** The self-exclusion guard dropped a reference doc only when its resolved path equalled the target's (`pth == target_abs`). A `--manifest` row carrying inline `text` (via `_load_reference_manifest`) resolves to `path=None`, so an inline **copy** of the target was never dropped — the target then reconstructed itself from its own copy and `coverage`/`originality` trivially collapsed to `1.0`/`0.0` (the self-positioning failure mode). A content fingerprint — `sha256` over a `\x1f`-joined serialization of the matcher's **canonical token stream** (`_tokens(text)`: lowercased `[a-z0-9]+` runs, the exact unit DJ-Search matches over) — now self-excludes any pool entry whose token stream equals the target's, **alongside** the existing path check (path OR content → exclude). Fingerprinting in the matcher's own equivalence class is what makes the guard sound: the matcher is case- AND punctuation-insensitive, so a case-only **or** punctuation-only variant of the target (e.g. `"alpha, beta…"` vs `"alpha beta…"`) reconstructs it span-for-span and must be dropped. An earlier draft fingerprinted under `normalize_for_char_ngrams` (lowercase + collapse-whitespace + strip), which **preserves punctuation** and so left a punctuation-only inline copy in the pool — the target still reconstructed itself, `coverage` 1.0 (Codex round-2 P1); the token-stream fingerprint closes that. The drop is fail-closed (a content match only ever drops an entry, never re-admits one; if exclusion empties the pool the surface routes through its existing empty-pool `bad_input` abstain path), and the count is surfaced through the existing `n_dropped_self` warning plus `results.assumptions.n_dropped_self` (honesty path). Sibling of the Codex P1 fixed in `cross_doc_novelty_profile.py` (content-fingerprint self-exclusion). Pinned by four regression tests in `test_originality_audit.py` (inline copy excluded; whitespace/case variant still caught; **punctuation-only variant still caught**; exclusion-to-empty routes to `bad_input`). No schema, envelope, capability, or `claim_license` change.
