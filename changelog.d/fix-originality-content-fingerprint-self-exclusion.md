### Fixed

**`originality_audit` no longer lets the target reconstruct itself from an inline-text copy in the reference pool (path-only self-exclusion hole).** The self-exclusion guard dropped a reference doc only when its resolved path equalled the target's (`pth == target_abs`). A `--manifest` row carrying inline `text` (via `_load_reference_manifest`) resolves to `path=None`, so an inline **copy** of the target was never dropped — the target then reconstructed itself from its own copy and `coverage`/`originality` trivially collapsed to `1.0`/`0.0` (the self-positioning failure mode). A content fingerprint — `sha256` over `stylometry_core.normalize_for_char_ngrams(text)` (lowercase + collapse-whitespace + strip) — now self-excludes any pool entry whose normalized content equals the target's, **alongside** the existing path check (path OR content → exclude), so a whitespace/case-variant copy is caught too. The drop is fail-closed (a content match only ever drops an entry, never re-admits one; if exclusion empties the pool the surface routes through its existing empty-pool `bad_input` abstain path), and the count is surfaced through the existing `n_dropped_self` warning plus `results.assumptions.n_dropped_self` (honesty path). Sibling of the Codex P1 just fixed in `cross_doc_novelty_profile.py` (content-fingerprint self-exclusion). Pinned by three regression tests in `test_originality_audit.py` (inline copy excluded; normalized whitespace/case variant still caught; exclusion-to-empty routes to `bad_input`). No schema, envelope, capability, or `claim_license` change.
