### Added

**`biber_features` — Biber lexico-grammatical register feature family for `voice_coherence` (spec neurobiber-v2, M1).** Adds a new optional `biber_features` family to the `stylometry_core` per-document feature pipeline alongside `pos_trigrams` / `dependency_ngrams`. The family emits a 96-feature named panel of Biber register rates (past-tense verbs, nominalizations, attributive adjectives, first-person pronouns, agentless passives, hedges, amplifiers, ...) — the Neurobiber panel ([arXiv:2502.18590](https://arxiv.org/abs/2502.18590), ~56x faster than rule-based Biber taggers, AV-competitive). The new `biber_features.py` module carries `BIBER_FEATURE_SCHEMA` (96 `(feature_id, label)` pairs, fixed order, single source of truth), `biber_family_features(vector)` (normalizer: fills missing → 0.0, silently drops unknowns, fixes order), a standalone `biber_panel` surface (the `dependency_distance_audit` abstain shape), and the injectable tagger seam.

**M1 (this build; model-free, CI):** schema + normalizer + the threaded `include_biber=False / biber_vector / biber_tagger` parameters through `extract_features` / `extract_entry_features` / `compare_to_baseline` / `build_profile` + `FIXED_FAMILIES` / `FAMILY_WEIGHTS` table edits + standalone panel + envelope + posture — all exercised over an injected/stub vector with no model importable. `include_biber` defaults to `False`; **existing `voice_distance` / `voice_profile` envelopes are byte-identical** (OFF-by-default invariant, downstream drift gate protected for apodictic + setec-voicewright). New `--include-biber` CLI flag on `voice_distance.py` and `voice_profile.py` opts in.

**M2 (deferred):** the real Neurobiber tagger behind a lazy `torch` / `transformers` import (`voice_fingerprint._LUAREncoder._load` pattern). No envelope-shape change in M2.

**Dual-use boundary:** Biber features separate human-vs-LLM AND LLM-vs-LLM ([arXiv:2410.16107](https://arxiv.org/abs/2410.16107)) — that inference is **refused** by the claim license, which explicitly names that paper and refuses model-family attribution, AI/human provenance, and authorship. The panel is descriptive; `model_family_attribution.py` does NOT import `biber_features`. Ships with 30+ stdlib tests (no `skipif`, all CI-blocking), including the AC4 no-verdict `_FORBIDDEN_KEYS` / `_walk_keys` recursive walk, AC10 opt-in-OFF drift-gate assertions in `test_voice_distance_schema.py` + `test_voice_profile_schema.py`, and AC11 stub-determinism + precomputed-vector parity.
