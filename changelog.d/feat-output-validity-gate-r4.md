### Changed

**R4 output-validity bounds gate at the `build_output()` boundary (`output_schema`).** Cheap, surface-agnostic plausibility checks now run over the `results` payload of every available envelope so an out-of-range COMPUTED value (the DirectML-surprisal class of bug — a confident, out-of-bounds number) raises `OutputValidityError` instead of shipping: any numeric that is NaN/inf is rejected; a named cosine-**similarity** field must be ∈ [−1, 1]; a surprisal / perplexity / entropy field must be ≥ 0 (no upper bound — `log₂|vocab|` is unknown at this boundary and is not invented); a probability field must be ∈ [0, 1]. Cosine **distance** and any field whose correct bound is not unambiguous are deliberately left unchecked. The gate is on by default and bypassable with `build_output(..., validate_bounds=False)`; `available=False` envelopes skip it (they legitimately carry partial data). An `OutputValidityError` from a surface script surfaces through the `setec_run.py` dispatcher as an R3 `internal_error` envelope.
