### Fixed

**`rank_turbulence_audit` no longer lets the target reconstruct itself into its own baseline through an inline-text copy or a content copy stored at a different path (path-only self-exclusion hole).** The baseline loader (`_load_baseline`) dropped a doc only when its resolved path equalled the target's, and — worse — a `--manifest` row carrying inline `text` was appended to the baseline with **no** self-exclusion check at all. An inline copy of the target (or an exact copy of the target stored at a different file path) therefore reconstructed the target into its own baseline, corrupting the rank-turbulence signal (`rtd` self-positioning collapse). A content fingerprint — `sha256` over the surface's **own** canonical token stream (`"\n".join(_TOKEN.findall(text.lower()))`, i.e. the same `[a-z]+`-over-lowercased-text tokenization `_counts` consumes, taken over the full vocabulary *before* the function-word filter) — now self-excludes any baseline entry whose content equals the target's **under RTD's matcher**, **alongside** the existing path check (path OR content → drop) across all three branches: the `--reference-dir`/`--baseline-dir` glob, the manifest inline-`text` row, and the manifest `text_path` row. Fingerprinting the actual token stream (not `stylometry_core.normalize_for_char_ngrams`, which only lowercases / collapses whitespace / strips and so PRESERVES punctuation and hyphens) closes a residual hole: because RTD compares `[a-z]+` word counts, a copy differing from the target only by punctuation (`"We, are here; and they"` vs `"We are here and they"`) or by hyphen-vs-space (`"state-of-the-art"` vs `"state of the art"`) tokenizes to an identical stream with identical counts and RTD 0.0 — yet the old `normalize_for_char_ngrams` fingerprint differed, so the punctuation-/hyphen-only copy slipped past the content guard and self-positioned. The stream is joined with a non-`[a-z]` separator (`"\n"`), so re-segmentation cannot collide (`"ab c"` ≠ `"a bc"`). Whitespace/case-variant copies remain caught (lowercasing + tokenization subsumes them). The drop is fail-closed (a content match only ever drops an entry, never re-admits one; if exclusion empties the baseline the surface routes through its existing empty-baseline `bad_input` abstain path), and the count is surfaced through the existing `dropped_self` warning plus `results.assumptions.dropped_self` (honesty path). Sibling of the Codex P1 fixed in `cross_doc_novelty_profile.py` / `originality_audit.py` / `corpus_novelty_audit.py` (content-fingerprint self-exclusion). Pinned by regression tests in `test_rank_turbulence_audit.py` (inline copy dropped; normalized whitespace/case-variant inline copy still caught; file copy at a different path dropped via content match; **punctuation-only and hyphen-vs-space copies dropped in both default and `--all-words` modes; separator-safe stream**). No schema, envelope, capability, or `claim_license` change.
