### Fixed

**`rank_turbulence_audit` no longer lets the target reconstruct itself into its own baseline through an inline-text copy or a content copy stored at a different path (path-only self-exclusion hole).** The baseline loader (`_load_baseline`) dropped a doc only when its resolved path equalled the target's, and — worse — a `--manifest` row carrying inline `text` was appended to the baseline with **no** self-exclusion check at all. An inline copy of the target (or an exact copy of the target stored at a different file path) therefore reconstructed the target into its own baseline, corrupting the rank-turbulence signal (`rtd` self-positioning collapse). A content fingerprint — `sha256` over `stylometry_core.normalize_for_char_ngrams(text)` (lowercase + collapse-whitespace + strip) — now self-excludes any baseline entry whose normalized content equals the target's, **alongside** the existing path check (path OR content → drop) across all three branches: the `--reference-dir`/`--baseline-dir` glob, the manifest inline-`text` row, and the manifest `text_path` row. A whitespace/case-variant copy is caught too. The drop is fail-closed (a content match only ever drops an entry, never re-admits one; if exclusion empties the baseline the surface routes through its existing empty-baseline `bad_input` abstain path), and the count is surfaced through the existing `dropped_self` warning plus `results.assumptions.dropped_self` (honesty path). Sibling of the Codex P1 fixed in `cross_doc_novelty_profile.py` / `originality_audit.py` / `corpus_novelty_audit.py` (content-fingerprint self-exclusion). Pinned by three regression tests in `test_rank_turbulence_audit.py` (inline copy dropped; normalized whitespace/case-variant inline copy still caught; file copy at a different path dropped via content match). No schema, envelope, capability, or `claim_license` change.
