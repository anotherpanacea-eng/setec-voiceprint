### Fixed

**`--include-biber` no longer turns into an uncaught traceback when the `neurobiber` package is installed (`biber_features` / `voice_distance` / `voice_profile`).** The round-1 fix wired a `missing_dependency` CLI guard that calls `biber_features._try_load_real_tagger()`, but that helper raised `NotImplementedError` the moment `import neurobiber` succeeded — the M2 NeurobiberTagger adapter is deferred, so a *present* package is a not-yet-integrated state, not a usable tagger. Because `NotImplementedError` is not an `ImportError`, it escaped the helper's `except ImportError` handler and propagated out of the guard, so **installing the very package named in the missing-dependency envelope flipped the clean `rc=3` abstention into an uncaught traceback** in all three CLIs (reproduced with a stub `neurobiber` in `sys.modules`). `_try_load_real_tagger()` now returns `None` unconditionally while the M2 adapter is deferred — it no longer probes `import neurobiber` at all — so `--include-biber` emits `available:false` / `reason_category=missing_dependency` / `rc=3` whether or not the dependency is importable. Pinned by package-present regressions in `test_biber_features.py` (`TestPackagePresentDoesNotCrash`), `test_voice_distance_schema.py`, and `test_voice_profile_schema.py` that inject a stub `neurobiber` and assert the clean envelope instead of a crash. No capability, schema, envelope, or `plugin.json` change (the abstain envelope is unchanged; only the crash-vs-abstain branch is fixed).
