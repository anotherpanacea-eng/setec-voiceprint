### Added

**`paraphrase_robustness.py` — detector-level AUC-degradation harness (spec 33, M1).** A new `validation`-surface tool (capability id `paraphrase_robustness`) that measures, for a labeled machine/human corpus, how much each detector's `P(machine > human)` separation **degrades** when the machine windows are paraphrased. It is the corpus-level / population-ranking counterpart of `paraphrase_ladder`'s per-(signal × rung) single-text decay curve — the two measure at different levels and do not overlap. M1 ships model-free: the orchestration (attack application + AUC/Δ/operating-point math + report assembly) runs over **injected** stage-2 detector scores, with **no model imported at module load**; the GPU re-scoring over the existing `binoculars_audit` / `surprisal_backend` / `fast_detect_curvature` / `variance_audit` arsenal is the M2 seam, in a later PR.

Per (detector × rung) the harness emits the discriminative `AUC` (WMW-U, computed by the existing `validation_harness.fallback_roc_auc`, reused not re-implemented, **oriented by each detector's known machine-vs-human sign** so a "machine lower" signal — Binoculars cross-perplexity, `surprisal_mean`/`surprisal_sd`, `burstiness_B`, `mtld` — is not read as inverted), `TPR` at FPR budgets {0.05, 0.10}, and the Δ from the unattacked rung 0. The sign/direction map (`DETECTOR_DIRECTION`, copied with provenance from `surprisal_backend.SMOOTHED_DIRECTION` and the `binoculars_audit` band) is pinned in `test_sign_direction_pinned` — silent inversion is the detection family's shared failure mode and now fails a test rather than shipping a flipped AUC.

Two REVIEW change-requests are folded into the code (the rest are M2-runbook framing, recorded in the spec): the runner defines its **own** injectable `Paraphraser` Protocol (REVIEW §1 verified the `LadderParaphraser` seam the roadmap spec assumed does not exist on `feat/raid-dipper-robustness`), and a rung-0 **scoring-divergence gate** (`test_rung0_auc_consistency`) asserts the harness's rung-0 AUC reproduces the AUC computed directly from the same injected scores — catching a pipeline divergence before any degradation number is read.

De-dup note (re-verified against this branch's `main`): `paraphrase_ladder` / `build_proxy_ladder` / the RAID `synonym_swap` transforms / `_SADASIVAN_CEILING` live ONLY on the unmerged `feat/raid-dipper-robustness` branch (spec 16), not on `main` — the `.pyc` left in `scripts/calibration/__pycache__/` is a stale artifact. So the harness does **not** import them; it defines a self-contained deterministic stdlib lexical proxy (`StdlibProxyParaphraser`, label `proxy_stdlib`) honestly weaker than a neural paraphraser. When raid-dipper lands, its `build_proxy_ladder` can be injected through the same Protocol; nothing here is re-implemented.

Posture (the `pan_replay` / `paraphrase_ladder` line, carried forward): the surface emits the per-(detector × rung) AUC/TPR/Δ table but **no** aggregate robustness or accuracy scalar — no `robustness_score` / `auc_retained` / `area_under_decay` / `is_robust` / `overall_robustness` / `headline` anywhere in the serialized payload (the banned-key `_walk` test enforces it at every depth; the legitimate per-cell `auc`/`tpr_at_fpr05` keys are the deliverable, not the banned summary scalar). Human windows are the fixed reference class and are never paraphrased (only the adversary's own machine output is attacked; pinned by `test_human_windows_not_paraphrased`). It sets no threshold, feeds no calibration or selector, and is never a reward. A corruption guard skips any rung whose paraphrase collapses a window below 50% of its length (never scores corrupted text). The `ClaimLicense` **refuses** any "robust to paraphrase" claim and any detector-accuracy headline, quoting Sadasivan's human/AI separability ceiling verbatim (Sadasivan et al. 2023, arXiv:2303.11156); attack-corpus texts are flagged as ineligible for voicewright training / humanization targets. `calibration_status: heuristic`. The cited Binoculars "~36% accuracy drop" figure is carried as a LEAD (M0-provisional: it is an **accuracy** drop under RAID **synonym substitution**, a strictly weaker attack than a neural paraphrase) and is asserted as fact nowhere — no test pins it.

A `feat` (MINOR): one new capability (`paraphrase_robustness`) on the existing `validation` surface (no new task-surface label). Roots: Sadasivan et al. (arXiv:2303.11156), DIPPER (arXiv:2303.13408), Binoculars (arXiv:2401.12070).
