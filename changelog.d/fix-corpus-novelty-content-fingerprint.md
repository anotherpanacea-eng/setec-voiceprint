### Fixed

**`corpus_novelty_audit` — content-fingerprint self-exclusion for inline duplicate docs (third sibling of the cross_doc_novelty_profile / originality_audit Codex P1).** The leave-one-out per-document self-exclusion loop dropped the exact index (`j==i`) and PATH-duplicates (`abs_j == abs_i`, a doc appearing twice at the same resolved path) but NOT CONTENT-duplicates: an inline-`text` manifest row (resolved_path `None`) carrying a copy of document `i` sat at a different index and was never dropped, so document `i` reconstructed itself from its own inline copy — trivially collapsing that doc's novelty to 0.0 and violating the surface's stated contract ("a doc never reconstructs itself"). The existing path-dup drop already proved the intent was to drop copies of the doc; the content gap was an incomplete implementation of that same intent. Added a `_content_fingerprint()` helper (sha256 over `stylometry_core.normalize_for_char_ngrams` — the reused normalizer, no new normalizer invented) and extended the drop condition to **index OR path OR content** in both the per-document leave-one-out loop and the mutual-reconstructibility census (fail-closed: a content match only ever drops a doc, never re-admits one). The `total_dropped_self` / `assumptions.dropped_self` honesty count and the self-exclusion warning now reflect content-dropped duplicates too. No new TASK_SURFACE, no capabilities change, no schema/envelope/claim_license change. Three pre-existing tests that relied on an exact-copy doc being retained to reconstruct another were updated to use a superset doc (target span + a distinct unique tail) — mirroring the originality_audit sibling fix — so they still exercise the span/coverage/top-source behavior without an exact self-copy.
