## Motivation

SETEC's current model-based detection axes (`binoculars_audit` cross-perplexity ratio,
`fast_detect_curvature` conditional curvature, `surprisal_audit` per-token moments) all measure
*how predictable a text's tokens are under an LLM at the natural ordering*. A sophisticated
humanizer that forces the text into higher-perplexity regions defeats them; Binoculars loses
accuracy under synonym substitution.

What none of them ask: **does this text's structure depend on the ordering of its parts?** Human
prose draws meaning from syntactic and narrative flow; permuting sentences or words should destroy
that meaning and spike perplexity under any LLM. AI prose, generated by next-token prediction, is
locally coherent in a different way — its inter-sentence dependencies are shallower, so shuffling
hurts it less *relative to itself*, yielding a smaller perplexity delta. This is a **structural,
order-dependence axis**: orthogonal in mechanism to the ratio/curvature/surprisal family
(perturbation probe, not prediction at the natural ordering), generator-agnostic by construction,
and robust by hypothesis to the synonym/paraphrase attack that hurts Binoculars (word-level
substitution does not change order-dependence structure).

## De-duplication and reuse statement

**Genuinely new capability** (confirmed against the live Code-Mac repo). No existing SETEC script,
surface, capability entry, or fleet spec covers perplexity-under-shuffling as a detection axis:

- `binoculars_audit.py` — cross-perplexity ratio between two models at the natural ordering. No
  shuffling.
- `fast_detect_curvature.py` (spec 03) — curvature of a single model's conditional log-prob surface
  at the natural ordering. No shuffling.
- `surprisal_audit.py` — per-token surprisal moments (mean / sd / lag-k ACF) at the natural
  ordering. No shuffling.
- `variance_audit.py` (glass-box) — surface burstiness / variance. No model, no shuffling.
- All `shuffle` hits in the codebase are data-partitioning RNG (shard_runner, calibration,
  sharding) or a watermark-partition PRF — mechanically unrelated.

The shuffle (sentence + word) and 5-feature extraction is new Python that runs CPU-only and reuses
the existing `surprisal_backend.SurprisalBackend` forward-pass API for the M2 model path.

**tocsin / specdetect-lastde note (REVIEW Rec-3).** Both seams (if ever built) call
`SurprisalBackend.score_text` at the natural ordering; the `score_window` orchestrator must not
cache model state between calls in a way that would interact with those callers. The current
orchestrator holds no cross-call state.

## Method

### Inputs and outputs

- **Input:** a prose text window (UTF-8 string; length floor ~50 words for stable perplexity).
- **Scorer model (M2 only):** a single causal LM (GPT-Neo-2.7B, HF id `EleutherAI/gpt-neo-2.7B`),
  loaded via `surprisal_backend.SurprisalBackend`. Fallback `gpt2_medium` (on-rig). **GPT-Neo-2.7B
  is added to `MODEL_ALIASES` only at M0/M2, gated behind CPU-first verification — NOT in this M1
  build.** M1 ships against an injected (stub) backend.
- **Output:** 5 scalar features per window, the raw perplexity triple (orig, sentence-shuffled,
  word-shuffled), the seed, and the score version, wrapped in `output_schema.build_output()`.

### Three forward passes (REVIEW R1 — expands the ROADMAP's two-pass framing)

The DOSSIER ROADMAP describes the experiment as "two forward passes per window (original +
shuffled)." **This spec implements three passes:** `ppl_orig` (natural ordering), `ppl_sent`
(sentence-shuffled), `ppl_word` (word-shuffled). Justification: sentence-order dependence and
local word-order dependence are **distinct structural probes**, and using both lets the feature set
decompose inter-sentence coherence from within-passage word coherence. The cost note ("3 forward
passes per window") and thermal math (3 × 212 = 636 passes on the PC rig) account for this. M1 is
unaffected (no model loads); the expansion is documented here per R1.

### Algorithm (clean-room; no vendored code)

1. **`ppl_orig`** — perplexity of the input under the scorer. `SurprisalBackend.score_text` returns
   per-token surprisal in **bits**; mean-bits → perplexity via `2 ** mean_bits`
   (`= exp(mean_nats)`). The M1 orchestrator accepts injected perplexity directly so no model is
   needed in CI.
2. **Shuffle variants** (seeded, default `--seed 0`, reported in output):
   - `sent_shuffled`: split on sentence boundaries (spaCy `en_core_web_sm` sentencizer if available;
     regex fallback `(?<=[.!?])\s+`), permute the sentences, rejoin.
   - `word_shuffled`: a single **passage-level** permutation of all whitespace-split tokens
     (REVIEW Rec-1 / Open Question #4 resolved: passage-level, not per-sentence — recorded in the
     `shuffle_words` docstring). Rejoin.
3. **`ppl_sent`, `ppl_word`** — perplexity of each shuffled variant under the same scorer.
4. **5 features** (REVIEW verified the arithmetic by hand):
   ```
   ppl_sum        = ppl_sent + ppl_word                       # total shuffled perplexity mass
   ppl_diff       = (ppl_sent + ppl_word) / 2 - ppl_orig      # average delta (signed)
   ppl_ratio      = ((ppl_sent + ppl_word) / 2) / ppl_orig    # relative lift
   ppl_log_ratio  = log(ppl_ratio)                            # log-scale lift
   ppl_pct_change = (ppl_ratio - 1) * 100                     # %-change (human-readable)
   ```
   **Sign / direction (pinned in a test).** Higher `ppl_diff` / `ppl_ratio` / `ppl_log_ratio` /
   `ppl_pct_change` = a larger jump under shuffling = **more order-dependent = more human-like**
   (the hypothesis direction). Silent sign inversion is the perturbation family's shared failure
   mode, so the direction is pinned by `test_feature_direction_sign`. The direction is the
   *hypothesis* — it is **not** a verdict and ships **uncalibrated**; the SETEC experiment confirms
   it empirically.

   **`ppl_sum` (REVIEW Rec-2 / S3).** Retained as a raw diagnostic mass, not as an independent
   discriminator. Confirm at M0 whether the paper's Table 1 names it; if not, it stays a diagnostic
   raw value rather than a claimed discriminative feature. M1 emits it; no number is asserted.
5. **Emit** — feature dict + `ppl_orig`, `ppl_sent`, `ppl_word`, `model_id`, `seed`, `score_version`
   (`"structural_shuffle_v1"`), wrapped in `build_output()` with the `ClaimLicense` block. No
   threshold, no verdict band, and no carried band value; the uncalibrated posture is surfaced via
   the `no_calibrated_thresholds_supplied` caveat.

1. `test_split_sentences` — 3-sentence fixture → 3 items (spaCy and regex fallback).
2. `test_shuffle_sentences_deterministic` — same seed same order; different seed differs (≥3 sents).
3. `test_shuffle_words_deterministic` — same seed same word order; different seed differs.
4. `test_shuffle_sentences_same_words` / `test_shuffle_words_same_words` — same multiset of words.
5. `test_feature_extraction_arithmetic` — `(2.0, 3.0, 4.0)` → `ppl_diff=1.5`, `ppl_ratio=1.75`,
   `ppl_log_ratio=log(1.75)`, `ppl_pct_change=75.0`, `ppl_sum=7.0` (6 d.p.).
6. `test_feature_direction_sign` — a larger shuffled perplexity yields strictly larger `ppl_diff` /
   `ppl_ratio` / `ppl_log_ratio` / `ppl_pct_change` (pins the human-likely direction; guards the
   family's silent-inversion failure mode).
7. `test_envelope_shape` — `task_surface`, `tool`, `results.features`, `results.ppl_orig`,
   `results.model_id`, `results.seed`.
8. `test_no_default_verdict_band` — no `band` / `verdict_band` key carried at all (no `"uncalibrated"` value either); no shipped threshold.
9. `test_no_verdict_keys` — recursive key walk finds no `is_ai` / `is_human` / `verdict` / `score` /
   `selection` / etc.
10. `test_claim_license_refuses_verdict` — `does_not_license` contains "verdict".
11. `test_claim_license_esl_failure_mode` — `does_not_license` contains "ESL" / "non-native".
12. `test_missing_torch_graceful` — torch absent → clean install hint (mirrors curvature).
13. `test_capabilities_entry_present` — drift linter green; manifest carries the entry.

15. `test_orthogonal_to_binoculars` — no `binoculars_score` / `cross_perplexity` /

16. Edge cases — empty input, single sentence (degenerate shuffle), tie / saturated perplexity,
    `ppl_orig == 0` (rank-0 → ratio guard).
